# AI 기반 DDoS 방화벽 프로젝트 계획서

## 1. 프로젝트 개요

본 프로젝트의 목적은 외부 네트워크에서 유입되는 트래픽을 실시간으로 분석하여 DDoS 공격 여부를 판단하고, 정상으로 분류된 트래픽만 내부 서버망으로 전달하는 AI 기반 방화벽 시스템을 구현하는 것이다.

일반적인 방화벽은 사전에 정의된 규칙이나 포트 정책을 기준으로 트래픽을 제어한다. 그러나 DDoS 공격은 짧은 시간 동안 대량의 요청을 발생시키거나, 정상 요청과 유사한 형태로 트래픽을 구성하여 단순 규칙만으로는 구분이 어려운 경우가 있다. 본 프로젝트는 이러한 문제를 해결하기 위해, 네트워크 패킷을 실시간으로 수집하고 일정 시간 단위의 흐름 정보로 변환한 뒤, 인공지능 모델을 이용해 공격 여부를 판별하는 구조를 제안한다.

즉, 이 프로젝트는 단순한 분류 모델 실험이 아니라, 실제 네트워크 환경을 가정한 보안 시스템 구현을 목표로 한다.

## 2. 프로젝트 배경 및 필요성

최근 서비스 거부 공격은 특정 서버나 네트워크 자원에 과도한 트래픽을 집중시켜 정상 사용자의 접근을 방해하는 대표적인 보안 위협으로 인식되고 있다. 특히 DDoS 공격은 여러 출발지에서 동시에 대량 트래픽을 발생시키기 때문에, 단일 출발지 차단이나 고정 임계치 방식만으로는 충분히 대응하기 어렵다.

또한 실제 네트워크 환경에서는 순간적인 정상 트래픽 증가와 공격 트래픽이 비슷하게 보일 수 있다. 따라서 단순 규칙 기반 탐지만으로는 오탐이 발생할 가능성이 있고, 반대로 공격을 정상으로 오판할 가능성도 존재한다.

이러한 배경에서 본 프로젝트는 다음과 같은 필요성을 가진다.

- 실시간 트래픽을 자동으로 분석할 수 있어야 한다.
- 정상 트래픽과 공격 트래픽을 데이터 기반으로 구분할 수 있어야 한다.
- 탐지 결과를 실제 네트워크 전달 제어와 연결할 수 있어야 한다.
- 연구용 분류 실험이 아니라, 시연 가능한 보안 시스템 형태로 구성되어야 한다.

## 3. 프로젝트 목적

본 프로젝트의 구체적인 목적은 다음과 같다.

첫째, 외부 네트워크에서 들어오는 패킷을 실시간으로 수집할 수 있는 환경을 구축한다.

둘째, 수집한 패킷을 일정 시간 단위로 집계하여 DDoS 탐지에 적합한 흐름 기반 feature를 생성한다.

셋째, 생성된 feature를 입력으로 사용하는 AI 모델을 학습시켜 정상 트래픽과 공격 트래픽을 이진 분류한다.

넷째, 탐지 결과에 따라 정상 트래픽만 내부 서버로 전달하고, 공격으로 판별된 트래픽은 차단하는 방화벽 동작을 구현한다.

다섯째, 전체 탐지 및 차단 과정을 로그와 시연 화면으로 확인할 수 있는 형태로 구성한다.

## 4. 프로젝트 문제 정의

이 프로젝트가 해결하려는 핵심 문제는 다음과 같다.

외부 네트워크에서 들어오는 트래픽 중 일부는 정상 서비스 요청이고, 일부는 DDoS 공격일 수 있다. 그러나 개별 패킷 하나만 보고는 이를 정확히 구분하기 어렵다. 공격 여부는 단일 패킷이 아니라 짧은 시간 동안의 패킷 흐름, 패킷 수의 증가 양상, 특정 프로토콜 비율, 출발지 분산도, 패킷 길이 분포 등의 정보와 더 밀접하게 관련된다.

따라서 본 프로젝트에서는 문제를 다음과 같이 재정의한다.

“실시간으로 수집되는 네트워크 패킷들을 일정 시간창 단위의 흐름 정보로 변환하고, 이를 입력으로 하는 AI 모델을 통해 해당 흐름이 정상인지 DDoS 공격인지 판별한다.”

이 문제 정의는 실제 네트워크 운영 환경과 더 가깝고, 단순 CSV 분류보다 프로젝트 목적에 부합한다.

## 5. 시스템 목표

본 시스템은 다음 조건을 만족하는 것을 목표로 한다.

- 외부망과 내부 서버망 사이에서 동작할 것
- 실시간 또는 준실시간으로 트래픽을 분석할 것
- 정상과 공격을 이진 분류할 것
- 정상 트래픽만 내부 서버에 전달할 것
- 공격 트래픽은 차단하거나 폐기할 것
- 탐지 결과와 차단 내역을 기록할 것

## 6. 전체 시스템 구조

### 6.1 네트워크 구성

프로젝트는 다음과 같은 가상 네트워크 구조를 기준으로 한다.

```text
외부 네트워크 망 (Bridge)
   ->
방화벽망 (VM 1)
   - 패킷 수집
   - 흐름 집계
   - AI 탐지
   - 전달/차단 제어
   ->
서버망 (VM 2, NAT 또는 내부망)
   - 테스트 서버 운영
```

이 구조에서 VM 1은 단순한 중간 노드가 아니라, 실제로 트래픽을 관찰하고 분석하며 전달 여부를 결정하는 핵심 방화벽 역할을 수행한다. VM 2는 정상 요청을 받는 내부 서버 역할을 수행한다.

### 6.2 처리 흐름

시스템의 전체 처리 순서는 다음과 같다.

```text
패킷 수집
-> 시간창 단위 집계
-> feature 생성
-> AI 모델 예측
-> 정상: 내부 서버로 전달
-> 공격: 차단
-> 결과 로그 저장
```

## 7. 핵심 아이디어

본 프로젝트의 핵심 아이디어는 “패킷을 직접 분류하지 않고, 흐름 단위의 특징을 추출해 AI가 판단하도록 한다”는 점이다.

네트워크 패킷 1개는 매우 짧은 순간의 정보만 포함하므로, 그 자체만으로 공격 여부를 판별하기 어렵다. 예를 들어 TCP SYN 패킷 하나는 정상 연결 요청일 수도 있고, 공격의 일부일 수도 있다. 반면 일정 시간 동안 동일 목적지로 향하는 대량의 SYN 패킷, 비정상적으로 높은 패킷 발생 속도, 특정 포트에 집중되는 트래픽, 비정상적인 ACK 비율 등은 공격 징후로 해석할 수 있다.

따라서 패킷을 일정 시간 단위로 묶어 통계적 특징을 만들고, 이 특징을 기반으로 AI 모델이 정상/공격을 판별하도록 설계한다.

## 8. 입력 데이터와 feature 설계 방향

본 프로젝트는 새 주제로 독립적으로 진행되므로, 데이터와 feature도 현재 목표에 맞게 새롭게 설계한다.

입력 단위는 개별 패킷이 아니라 `1초` 또는 `수 초 단위 window`로 설정한다. 각 window마다 다음과 같은 feature를 계산하는 방향이 적절하다.

- 전체 패킷 수
- 전체 바이트 수
- 초당 패킷 수
- 초당 바이트 수
- 고유 source IP 수
- 고유 destination IP 수
- 고유 source port 수
- 고유 destination port 수
- TCP 패킷 비율
- UDP 패킷 비율
- ICMP 패킷 비율
- SYN 패킷 수
- ACK 패킷 수
- RST 패킷 수
- SYN/ACK 비율
- 평균 패킷 길이
- 패킷 길이 표준편차
- 평균 inter-arrival time
- destination port 집중도
- 특정 source-destination 쌍의 반복 빈도

이 feature들은 실제 패킷 캡처 상황에서 계산 가능해야 하며, 학습 단계와 운영 단계에서 동일한 형식으로 생성되어야 한다.

## 9. AI 모델 설계 방향

본 프로젝트는 정상과 공격을 구분하는 이진 분류 문제로 설정한다. 따라서 분류 모델은 탐지 성능과 실시간 추론 속도를 함께 고려해 선택해야 한다.

가장 적절한 초기 모델은 `RandomForest`이다. 그 이유는 다음과 같다.

- 구현이 비교적 단순하다.
- 다양한 수치형 feature를 안정적으로 처리할 수 있다.
- 학습과 추론이 비교적 빠르다.
- feature 중요도를 확인할 수 있어 결과 해석이 쉽다.
- 과제 발표에서 모델 구조를 설명하기 용이하다.

추가로 비교 실험이 가능하다면 `XGBoost`, `LightGBM`, `IsolationForest` 등을 후보로 둘 수 있다. 다만 프로젝트의 중심은 모델 비교 자체가 아니라, 실시간 탐지와 방화벽 동작 구현에 두는 것이 적절하다.

## 10. 학습 및 평가 방향

학습 단계에서는 정상 트래픽과 공격 트래픽을 모두 포함하는 데이터를 준비하고, 이를 동일한 feature schema로 변환한 뒤 모델을 학습시킨다.

평가 시 단순 accuracy만으로 성능을 판단하는 것은 적절하지 않다. DDoS 탐지에서는 공격을 놓치지 않는 것이 중요하며, 동시에 정상 트래픽을 과도하게 차단하지 않아야 한다. 따라서 다음 지표를 중심으로 평가해야 한다.

- Precision
- Recall
- F1-score
- Confusion Matrix
- False Positive Rate
- 추론 지연시간

특히 `Recall`은 공격 탐지율과 직접 관련되고, `False Positive Rate`는 정상 사용자의 서비스 차단 가능성과 관련되므로 중요한 평가 기준이 된다.

## 12. 개발 단계별 진행 방향

### 12.1 1단계: 환경 구성

- VM 1과 VM 2 네트워크 구조 설정
- 외부망, 방화벽망, 서버망 연결 확인
- 테스트용 서버 실행

### 12.2 2단계: 패킷 수집기 구현

- VM 1에서 패킷 캡처
- 프로토콜, 포트, 길이, 시간 정보 추출
- 수집 로그 저장

### 12.3 3단계: 흐름 집계 및 feature 생성

- 패킷을 1초 단위 window로 집계

- 학습/운영 공통 schema 정의

### 12.4 4단계: 모델 학습

- 정상/공격 데이터 준비
- feature 변환
- RandomForest 학습
- 평가 지표 확인

### 12.5 5단계: 실시간 추론 연동

- 실시간 패킷 수집 결과를 feature로 변환
- 모델에 입력하여 공격 여부 예측
- 예측 결과를 로그로 기록

### 12.6 6단계: 차단 기능 구현

- 공격으로 판별된 트래픽 차단
- 정상으로 판별된 트래픽 전달 유지
- 차단 정책 및 해제 정책 설정

### 12.7 7단계: 시연 및 검증

- 정상 접속 시 서버 응답 확인
- 공격 트래픽 발생 시 탐지 여부 확인
- 차단 후 서버 접근 차단 여부 확인
- 로그 및 결과 화면 정리

- 네트워크 데이터 처리와 AI 기반 분류를 결합한 융합형 프로젝트를 수행할 수 있다.
- DDoS 탐지 문제를 패킷 수준이 아닌 흐름 수준으로 해석하는 실용적 접근을 제시할 수 있다.
- 탐지 결과를 실제 차단 동작과 연결함으로써 보안 시스템의 활용 가능성을 보여줄 수 있다.

## 15. 프로젝트 진행 시 주의사항

첫째, 패킷 1개만으로 공격 여부를 판별하려는 접근은 피해야 한다. 반드시 시간창 기반 통계로 변환해야 한다.

둘째, 학습용 데이터와 실제 운영 데이터의 feature 구조가 다르면 모델을 바로 사용할 수 없다. 따라서 feature schema를 먼저 확정하는 것이 중요하다.

셋째, 정확도 수치만 높다고 좋은 모델은 아니다. 정상 트래픽을 과도하게 차단하지 않는지도 반드시 함께 검토해야 한다.

넷째, 시연 환경에서는 실제 대규모 공격을 재현하기 어렵기 때문에, 제한된 환경에서 확인 가능한 테스트 시나리오를 별도로 준비해야 한다.

## 16. 결론

본 프로젝트는 외부망과 내부 서버망 사이에 위치한 방화벽망에서 실시간 네트워크 트래픽을 수집하고, 이를 흐름 기반 feature로 변환한 뒤 AI 모델을 통해 DDoS 여부를 판별하는 시스템을 구현하는 것을 목표로 한다.

정상 트래픽만 내부망으로 전달하고 공격 트래픽은 차단함으로써, 단순한 분류 실험을 넘어 실제 동작 가능한 AI 기반 DDoS 방화벽의 형태를 구현할 수 있다. 또한 본 프로젝트는 네트워크 보안, 데이터 처리, 머신러닝, 시스템 제어를 하나의 구조로 통합한다는 점에서 과제 주제로서도 충분한 의미를 가진다.
